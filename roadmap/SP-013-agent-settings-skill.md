body: "# SP-013: Agent Settings Management Tool\n\n**Status:** \U0001F4CB Proposed  \n**Depends on:** SP-003  \n**Priority:** Medium  \n**Effort Estimate:** ~2-3 days\n\n## Problem\n\nUsers frequently ask the agent things like:\n- \"Change my model to claude-sonnet-4-20250514\"\n- \"Switch to OpenAI\"\n- \"Turn on PDF OCR\"\n- \"Set my commit provider to deepseek\"\n- \"What model am I using?\"\n- \"Does my OpenAI key work?\"\n\nThe agent has **no tool to read or modify configuration**. It can only tell users to open Settings or edit config files manually.\n\n## Design Principles\n\n1. **No skill needed** — Settings management is only relevant when the user asks. Unlike `project-planning` which is useful at project start, settings are an occasional request. A skill would waste context on 99% of conversations. The tool is self-documenting instead.\n\n2. **Self-documenting returns** — Invalid keys, invalid values, and unknown operations all return actionable guidance. The agent learns by calling, not by reading skill docs.\n\n3. **Useful over terse** — Every return should give the agent enough information to act without a follow-up call. If `set` changed something, confirm what changed and what the agent should consider doing next. If `get` returns a key, include its type and valid values when relevant. Don't pad, don't dump, but don't be stingy either.\n\n## Proposed Solution\n\n### Tool: `manage_settings`\n\n```go\nregistry.RegisterTool(ToolConfig{\n    Name:        \"manage_settings\",\n    Description: \"Read and modify Sprout agent configuration (provider, model, features). Use 'get' to inspect, 'set' to change, 'list_providers' to see available providers, or 'test_credential' to validate an API key.\",\n    Parameters: []ParameterConfig{\n        {\"operation\", \"string\", true, nil, \"get | set | list_providers | test_credential\"},\n        {\"key\", \"string\", false, nil, \"Setting key for get/set (e.g. 'provider', 'model', 'pdf_ocr_enabled')\"},\n        {\"value\", \"string\", false, nil, \"New value for 'set' operations\"},\n        {\"provider\", \"string\", false, nil, \"Provider name for test_credential\"},\n    },\n    Handler: handleManageSettings,\n})\n```\n\nThe tool definition is intentionally minimal — 4 short parameter descriptions. All guidance lives in the return values.\n\n### Operations\n\n#### `get` — Read Configuration\n\n**Without a key:** returns a summary of the current configuration that covers what users most commonly ask about.\n\n```\nmanage_settings(operation=\"get\")\n\n→ Current configuration:\n  Provider: openai\n  Model: gpt-4o\n  Reasoning effort: medium (valid: low, medium, high, \"\" for auto)\n  Disable thinking: false\n  Self-review gate: code (valid: off, code, always)\n  PDF OCR: enabled, provider=ollama, model=llama3.2-vision\n  Subagents: parallel enabled, max 2, provider=openai, model=gpt-4o-mini\n  Commit messages: provider=deepseek, model=deepseek-chat\n  Code review: provider=(inherits), model=(inherits)\n  Zsh command detection: false\n  Provider models:\n    openai → gpt-4o\n    deepseek → deepseek-chat\n    ollama → llama3.2\n```\n\nNote: type and valid-value hints are included inline for enum/bool fields so the agent knows what it can change without a second call.\n\n**With a key:** returns the value plus type context.\n\n```\nmanage_settings(operation=\"get\", key=\"provider_models\")\n→ provider_models (map of provider name → model name):\n  openai → gpt-4o\n  deepseek → deepseek-chat\n  ollama → llama3.2\n```\n\n```\nmanage_settings(operation=\"get\", key=\"subagent_max_parallel\")\n→ subagent_max_parallel: 2 (int, valid: 1–8)\n```\n\n**Invalid key:** returns guidance with the full list of valid keys.\n\n```\nmanage_settings(operation=\"get\", key=\"foo\")\n→ Unknown key \"foo\". Valid keys:\n  provider, model — switch live agent (session only)\n  last_used_provider — default provider (persisted)\n  provider_models.{name} — model per provider (persisted)\n  provider_priority — provider fallback order (persisted, JSON array)\n  reasoning_effort — low|medium|high|\"\" (persisted)\n  disable_thinking — true|false (persisted)\n  self_review_gate_mode — off|code|always (persisted)\n  pdf_ocr_enabled, pdf_ocr_provider, pdf_ocr_model (persisted)\n  subagent_max_parallel, subagent_parallel_enabled, subagent_provider, subagent_model (persisted)\n  commit_provider, commit_model, review_provider, review_model (persisted)\n  enable_zsh_command_detection, auto_execute_detected_commands (persisted)\n  resource_directory, system_prompt_text (persisted)\n  Use 'get' without a key to see current summary.\n```\n\nThis is longer but the agent only sees it once — when it guesses wrong. After that it knows the keys. And it's complete enough that no follow-up is needed.\n\n#### `set` — Change a Setting\n\nAll values are strings. The tool coerces automatically (`\"true\"` → bool, `\"4\"` → int).\n\n**Successful set:** confirm what changed, note persistence behavior.\n\n```\nmanage_settings(operation=\"set\", key=\"provider\", value=\"openai\")\n→ ✅ Provider switched to openai.\n  This is a session override — it will reset when the conversation ends.\n  To persist across sessions, also set: last_used_provider=openai\n  Live agent is now using openai.\n```\n\n```\nmanage_settings(operation=\"set\", key=\"pdf_ocr_enabled\", value=\"true\")\n→ ✅ pdf_ocr_enabled set to true (saved to global config).\n  PDF OCR is now enabled. Current OCR provider: ollama, model: llama3.2-vision.\n  To change the OCR model: set pdf_ocr_provider or pdf_ocr_model.\n```\n\n```\nmanage_settings(operation=\"set\", key=\"subagent_max_parallel\", value=\"4\")\n→ ✅ subagent_max_parallel changed from 2 to 4 (saved to global config).\n```\n\n**Invalid value:** explain what went wrong and list valid options.\n\n```\nmanage_settings(operation=\"set\", key=\"reasoning_effort\", value=\"maybe\")\n→ Invalid value \"maybe\" for reasoning_effort (string).\n  Valid values: low, medium, high, \"\" (auto)\n```\n\n```\nmanage_settings(operation=\"set\", key=\"subagent_max_parallel\", value=\"0\")\n→ Invalid value \"0\" for subagent_max_parallel (int).\n  Must be between 1 and 8.\n```\n\n**`provider` and `model` are special:** they switch the live agent immediately (session override, not persisted). The return value always explains the session-vs-persist distinction.\n\n**All other keys** persist to global config (`~/.config/sprout/config.json`).\n\n#### `list_providers` — Show Available Providers\n\n```\nmanage_settings(operation=\"list_providers\")\n\n→ Available providers:\n  openai       ✅ has API key   model: gpt-4o\n  deepseek     ✅ has API key   model: deepseek-chat\n  openrouter   ✅ has API key   model: (default)\n  deepinfra    ✅ has API key   model: (default)\n  ollama       (local)          model: llama3.2\n  lmstudio     (local)          model: (default)\n  ollama-turbo (local)          model: (default)\n  mistral      ❌ no API key    model: (default)\n  minimax      ❌ no API key    model: (default)\n  chutes       ❌ no API key    model: (default)\n  cerebras     ❌ no API key    model: (default)\n  zai          ❌ no API key    model: (default)\n  ✅ = has API key stored, ❌ = no API key, (local) = runs locally\n```\n\nOne line per provider. Shows credential status and current/default model. Complete enough for the agent to recommend a provider.\n\n#### `test_credential` — Validate API Key\n\n```\nmanage_settings(operation=\"test_credential\", provider=\"openai\")\n→ ✅ openai API key is valid.\n```\n\n```\nmanage_settings(operation=\"test_credential\", provider=\"mistral\")\n→ ❌ mistral API key is invalid or missing (HTTP 401: Unauthorized).\n  The stored key may have expired or been revoked. The user can update it in Settings.\n```\n\n### Self-Documenting Behavior\n\nEvery invalid input produces a helpful message that teaches the agent the valid options. This means:\n\n- **No upfront documentation needed** — the agent discovers valid keys/values through interaction\n- **First bad guess teaches everything** — an unknown key returns the complete list of valid keys with type hints\n- **Validation messages are the docs** — every enum field lists its valid values on invalid input\n- **Successful operations confirm and guide** — `set` confirms the change and mentions related keys the agent might want to adjust\n\nThis replaces the need for a skill. The agent's context is only populated with settings info when the user actually asks about settings.\n\n### Key Reference (for handler implementation)\n\nThe handler validates against this whitelist. return value\n- `system_prompt_text` requires explicit user intent (the agent should already have it from the user's message)\n- `test_credential` makes a real API call — the tool refuses to repeat it for the same provider within 60 seconds\n\n### Handler Implementation\n\n```go\n// pkg/agent/tool_handlers_settings.go (new file)\n\nfunc handleManageSettings(ctx context.Context, a *Agent, args map[string]interface{}) (string, error) {\n    operation, _ := args[\"operation\"].(string)\n    cm := a.configManager\n    if cm == nil {\n        return \"\", fmt.Errorf(\"configuration manager not available\")\n    }\n    switch operation {\n    case \"get\":\n        return handleSettingsGet(cm, args)\n    case \"set\":\n        return handleSettingsSet(a, cm, args)\n    case \"list_providers\":\n        return handleSettingsListProviders(cm)\n    case \"test_credential\":\n        return handleSettingsTestCredential(cm, args)\n    default:\n        return \"\", fmt.Errorf(\"unknown operation %q. Use: get, set, list_providers, test_credential\", operation)\n    }\n}\n```\n\nEach sub-handler is responsible for:\n1. Validating inputs against the whitelist/enums above\n2. Returning actionable guidance on invalid input (include type + valid values)\n3. Applying changes (global via `cm.UpdateConfig()`, session via `a.SetProvider()`/`a.SetModel()`)\n4.
