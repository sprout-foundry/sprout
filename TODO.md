### Phase 2: User-configurable policy with safe import/export
- [ ] SP-049-2a: Add `Shell` config section. In `pkg/configuration/config.go`, add a `Shell` struct to the config schema with fields: `UserSafePatterns []ShellPattern`, `UserDangerousPatterns []ShellPattern`, `WorkspaceOverlay WorkspaceOverlayConfig`. `ShellPattern` has `Match string`, `Kind string` ("prefix" or "regex"), optional `Reason string`. `WorkspaceOverlayConfig` has `Mode string` ("tighten_only" | "trusted" | "ignore", default "tighten_only"). Wire into the existing config load/save/merge paths. Acceptance: a config with a `shell` section loads without error; the struct is accessible from `AgentConfig`; existing configs without the section get zero-value defaults.
- [ ] SP-049-2b: Implement pattern resolution in the classifier. In `pkg/agent_tools/security_classifier.go`, after the built-in classification, consult user patterns per the spec's resolution order (critical patterns → built-in DANGEROUS → user DANGEROUS → user SAFE → built-in CAUTION → built-in SAFE). Longest matching pattern wins within a tier. User SAFE can only override built-in CAUTION (never built-in DANGEROUS or critical). User patterns are compiled once and cached. Acceptance: a `user_safe_patterns` entry for `my-deploy-script` classifies SAFE; a `user_safe_patterns` entry for `rm -rf /` is IGNORED and still hard-blocks (the hard invariant).
- [ ] SP-049-2c: Workspace overlay loader. Read `.sprout/shell-policy.json` from the workspace root. Honor the `workspace_overlay.mode` setting: `tighten_only` (default) loads only `user_dangerous_patterns` from the workspace and ignores `user_safe_patterns` with a warning; `trusted` loads the full overlay only if the file's SHA-256 matches a recorded hash in `~/.sprout/trusted-workspaces.json`; `ignore` never loads workspace policy. If a trusted file's hash changes, fall back to `tighten_only`. Acceptance: a workspace file with a `user_safe_patterns` entry is ignored under `tighten_only`; honored under `trusted` (after `policy trust`); hash change reverts to `tighten_only`.
- [ ] SP-049-2d: `sprout policy` CLI subcommand tree. Add a `policy` subcommand in `cmd/` with: `list` (human-readable effective policy), `dump [--format=json|yaml]` (full effective policy with source annotations: built-in/user-config/workspace/cli-override), `add safe|dangerous <pattern>`, `remove safe|dangerous <pattern>`, `export [--format=yaml]`, `import <file>` (merge with diff confirmation), `trust` (record workspace file hash), `untrust [--all]`. Acceptance: `policy add safe 'my-tool'` persists to config; `policy export | import` round-trips losslessly; `policy trust` records the hash and unlocks `trusted` overlay mode.
- [ ] SP-049-2e: Phase 2 tests. Test pattern resolution order (user SAFE overrides built-in CAUTION but not DANGEROUS; longest match wins; hard invariant for `rm -rf /` in user_safe_patterns). Test workspace overlay modes (tighten_only ignores safe patterns; trusted honors after trust + hash; hash change reverts). Test CLI commands (add/remove round-trip, export/import lossless, trust/untrust). Acceptance: all new tests pass; no regressions in existing classifier tests.
