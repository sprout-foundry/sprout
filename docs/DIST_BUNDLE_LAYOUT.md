# Sprout WebUI Dist Bundle Layout — Canonical Specification

**Status:** Canonical (SP-015-R6)
**Date:** 2026-06-18

## Overview

This document defines the canonical directory structure that the Sprout WebUI
distribution bundle (`dist/`) must produce. The bundle is consumed by:

1. **Foundry (workspace mode)** — embedded via Go `embed.FS` in
   `platform/internal/api/webui/`, served at `/webui/`
2. **Cloudflare Pages** — static hosting for the browser IDE
3. **Docker containers** — copied into the foundry image at build time

The `scripts/build-webui-dist.mjs` script must produce exactly this structure.

## Canonical Layout

```
dist/
├── index.html               # SPA entry point (Vite-generated, post-processed)
├── assets/                   # Vite build output (JS, CSS, fonts, images)
│   ├── index-[hash].js       # Main application bundle
│   ├── index-[hash].css      # Main stylesheet
│   ├── react-[hash].js       # Vendor chunk (React runtime)
│   ├── codemirror-[hash].js  # Vendor chunk (CodeMirror editor)
│   ├── SettingsPanel-[hash].js
│   ├── SettingsPanel-[hash].css
│   ├── onnxruntime-[hash].js # ONNX Runtime Web (lazy-loaded)
│   ├── codemirror-langs-[hash].js
│   ├── ReviewWorkspaceTab-[hash].js
│   └── *.wasm                # ONNX WASM binaries (if any)
├── wasm/                     # Go-compiled WebAssembly modules
│   ├── sprout.wasm           # Shell-only WASM (loaded on page load)
│   ├── embedding.wasm        # Embedding/memory WASM (lazy-loaded, SP-045-3)
│   ├── wasm_exec.js          # Go WASM runtime (from GOROOT)
│   └── version.json          # Build metadata (git tag, commit, date)
├── manifest.json             # PWA manifest
├── browserconfig.xml         # Windows tile config
├── icon-192.png              # PWA icon (192x192)
├── icon-512.png              # PWA icon (512x512)
├── logo-mark.svg             # Logo mark (dark)
├── logo-mark-light.svg       # Logo mark (light)
├── sw.js                     # Service worker (network-only)
├── placeholder               # Build placeholder (removed post-build)
├── .sprout-version           # Sprout version stamp
└── version.json              # Root-level version metadata (generated)
```

## Key Files

### `index.html`

The SPA entry point. Vite generates this with hashed asset references.
Post-processing by `build-webui-dist.mjs` may adjust the base path:

- **Local mode:** base `/` (default)
- **Cloud mode (Foundry embed):** base `/webui/` (SP-015 cloud integration)
- **Cloud mode (Cloudflare Pages):** base `/`

### `wasm/sprout.wasm`

The main Go→WASM binary. Compiled from `cmd/wasm/` with:

```
GOOS=js GOARCH=wasm go build -tags "grammar_blobs_external osusergo" -ldflags="-s -w" \
  -o dist/wasm/sprout.wasm ./cmd/wasm/
```

This is the **shell-only** module (SP-045-3). It does NOT include the
embedding/semantic-search code. Expected size: ~30-40MB (stripped).

### `wasm/embedding.wasm`

The lazy-loaded embedding module (SP-045-3). Compiled from
`cmd/embedding-wasm/`. The browser fetches this only when the user first
triggers semantic search or memory features.

```
GOOS=js GOARCH=wasm go build -tags "grammar_blobs_external osusergo" -ldflags="-s -w" \
  -o dist/wasm/embedding.wasm ./cmd/embedding-wasm/
```

### `wasm/wasm_exec.js`

The Go WASM runtime support file. Copied from `$GOROOT/lib/wasm/wasm_exec.js`.
Both `sprout.wasm` and `embedding.wasm` share this runtime.

### `wasm/version.json`

Build metadata generated by `scripts/build-wasm.sh`:

```json
{
  "version": "v1.0.0",
  "commit": "abc1234",
  "buildDate": "2026-06-18T10:00:00Z",
  "gitTag": "v1.0.0"
}
```

### `assets/`

Vite's output directory. The exact filenames include content hashes and change
on every build. The structure is:

- `index-[hash].js` — Main application bundle
- `index-[hash].css` — Main styles
- `[name]-[hash].js` — Code-split chunks (SettingsPanel, ReviewWorkspaceTab, etc.)
- `react-[hash].js` — React vendor chunk
- `codemirror-[hash].js` — CodeMirror vendor chunk
- `onnxruntime-[hash].js` — ONNX Runtime Web (lazy)
- `*.wasm` — ONNX WASM binaries (onnxruntime-web)

### `version.json` (root)

Root-level version metadata generated by `generateVersionJson()`:

```json
{
  "version": "v1.0.0",
  "commit": "abc1234",
  "buildDate": "2026-06-18T10:00:00Z",
  "mode": "cloud"
}
```

## Build Verification

The `build-webui-dist.mjs` script includes a `verifyWasmFiles()` step that
checks for required WASM files. The canonical set is:

- `wasm/sprout.wasm` — REQUIRED (shell module)
- `wasm/wasm_exec.js` — REQUIRED (Go runtime)
- `wasm/embedding.wasm` — OPTIONAL but expected (embedding module, SP-045-3)

## Notes

- The old `static/js/` layout (mentioned in SP-015 spec) is no longer used.
  Vite outputs to `assets/` by default. This is the current canonical layout.
- The `wasm/` subdirectory keeps Go WASM artifacts separate from Vite's
  JavaScript bundles.
- The `.sprout-version` and `placeholder` files are build artifacts that
  may be safely deleted post-build.
